# CSMA/CD
IEEE.802.3

## CSMA
- CSMA（Carrier Sense Multipe Access，载波监听多路访问）
- 基本原理：发送数据前先监听信道上是否有人在发送，如果有说明信道在忙，反之则说明信道空闲，然后根据预定策略决定：
（1）若信道空闲，是否立即发送。
（2）若信道在忙，是否继续监听。

## CSMA三种监听算法
（1）1-坚持型：只要信道变闲，就可以立即发送，如果冲突后退再试。
- 有利于抢占信道，减少信道空闲时间，但多个站同时都在监听信道时必然会发生冲突。
（2）P-坚持型：信道变闲时以概率P发送，否则延迟一个时槽。
- P的大小可调整。
（3）非坚持型：如果空闲，则发送，否则后退再试。
- 减少冲突概率，但因为后退导致信道闲置一段时间，使信道利用率降低，增加了发送时延。

## 冲突检测原理CD
- 载波监听只能**减小冲突概率，无法完全避免**。
- 若帧比较长或两个帧发生**冲突还能继续发送，会浪费网络带宽**。为了改进带宽的利用率，发送应采取**一边发送一边监听**的冲突检测方法，即：
  （1）发送期间同时接受，并把接受数据与站中存储的数据进行比较
  （2）若比较结果一致，那么没有冲突，重复（1）。
  （3）若比较结果不一致，说明发送冲突，**立即**停止发送，并发送一个**简短干扰信号（jamming）**，使所有站都停止发送。
  （4）发送jamming信号后，随机等待一段时间，重新监听再尝试发送。

- 带冲突检测的监听算法把**浪费带宽的时间减少到检测冲突的时间** 。
