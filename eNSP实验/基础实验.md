# 🌐 华为网络实战：从子网逻辑到跨VLAN路由 (eNSP 篇)
## 核心背景
在进行跨网段通信实验时，我们不只是在连线，更是在构建一套二层隔离、三层互通的体系。

一、 子网划分的“潜规则”
在设计网络时，掩码决定了广播域的大小。

复习点：只要 IP 落在子网掩码划分的“线”两侧，逻辑上就是隔离的。

实战建议：生产环境尽量避免使用 /32（主机路由）或 /30（极窄子网），推荐从 /29 或 /28 起步，为网关和未来扩展留出空间。

二、 重点攻克：VLAN 与 Vlanif 的缠斗
初学者最常问：划分 VLAN 就是划分子网吗？
结论：不是。VLAN 解决二层广播隔离（墙），Vlanif 解决三层路由互通（门）。一个 VLAN 只有配上对应的 Vlanif，才能实现跨网段通信。

## 二层交换机配置 (LSW)
二层交换机不负责路由，只需确保包能带着正确的“标签”传给网关。

Bash
# 1. 创建 VLAN
[LSW] vlan batch 10 20

# 2. 接口划分（接入层）
[LSW] interface Ethernet 0/0/1
[LSW-Ethernet0/0/1] port link-type access
[LSW-Ethernet0/0/1] port default vlan 10

# 3. 上联接口（发往网关）
[LSW] interface Ethernet 0/0/3
[LSW-Ethernet0/0/3] port link-type access
[LSW-Ethernet0/0/3] port default vlan 10
## 三层设备网关配置 (AR 路由器)
当物理接口无法直接配 IP（如 AR201 的 LAN 口）时，必须使用 Vlanif 绕路。

Bash
# 1. 先创建对应的 VLAN 实体
[AR] vlan 10

# 2. 进入逻辑接口配 IP（门牌号）
[AR] interface Vlanif 10
[AR-Vlanif10] ip address 192.168.1.1 24

# 3. 关联物理接口（让包能进门）
[AR] interface Ethernet 0/0/0
[AR-Ethernet0/0/0] port link-type access
[AR-Ethernet0/0/0] port default vlan 10
三、 避坑指南：华为 eNSP 那些“迷路”瞬间
## ip address 报错
现象：在接口视图输入 ip ? 报错，或者提示命令未识别。

真相：该接口处于二层交换模式。

绝招：输入 undo portswitch 强制切换为三层路由模式。如果设备（如 AR201 右侧口）硬件不支持切换，请改用上述 Vlanif 方案。

## 终端配置的“致命伤”
故障复盘：实验中 PC8 无法通信，PC9/10/11 正常。

原因：网关地址填错。

原理：PC 发现目标地址跨网段时，会寻找网关 MAC。网关错，则包不出网卡。

四、 实验复盘思维导图
接入层 (LSW)：vlan -> port link-type access -> port default vlan

核心层 (AR)：vlan -> vlanif -> ip address -> physical interface assignment

排错三部曲：

ping 127.0.0.1 (自检)

ping 网关IP (二层/三层边界检查)

ping 目标IP (全路径路由检查)

五、 给未来自己的寄语
网络工程师不仅是在敲命令，更是在设计流量的“水利工程”。

VLAN 是水库的隔板。

Vlanif 是连接不同隔板的水泵。

掩码 是划分水域的刻度。

博主贴士：
如果你在 AR201 上死活配不上 IP，记得看一眼接口名称。Ethernet 通常很顽固，而 GigabitEthernet 通常很温柔。实在不行，Vlanif 是你永远的后盾！

# 增补章节
  Access 与 Trunk —— 交换机的“交通规则”在多 VLAN 环境下，交换机接口必须明确自己的“身份”。如果配置错误，数据包就会像进了死胡同一样丢失。1. 核心定义Access (接入模式)：连接对象：PC、打印机等终端设备。特点：只属于一个 VLAN。动作：发往 PC 时会“剥离”VLAN 标签，因为 PC 只认原始数据帧。Trunk (干道模式)：连接对象：交换机与交换机、交换机与路由器。特点：允许多个 VLAN 流量通过。动作：转发时保留 VLAN 标签（802.1Q），确保对端设备知道这属于哪个 VLAN。2. 实验配置命令A. 配置 Access 端口 (连 PC)Bash[LSW] interface Ethernet 0/0/1
[LSW-Ethernet0/0/1] port link-type access        # 1. 定性：我是接入模式
[LSW-Ethernet0/0/1] port default vlan 10        # 2. 定量：我只属于 VLAN 10
## 配置 Trunk 端口 (连交换机/路由器)Bash[LSW] interface GigabitEthernet 0/0/1
[LSW-GigabitEthernet0/0/1] port link-type trunk              # 1. 定性：我是干道模式
[LSW-GigabitEthernet0/0/1] port trunk allow-pass vlan 10 20  # 2. 授权：放行 VLAN 10 和 20
### 提示：若需放行所有 VLAN，可用 vlan all，但生产环境建议按需授权更安全。
 【**重点复习**】数据包的“贴标与撕标”过程为了方便复习，我们追踪一个包的旅程：进入交换机：PC 发出原始包 $\rightarrow$ 进入 Access 口 $\rightarrow$ 交换机给它打上 VLAN 10 的标签。跨机传输：标签包进入 Trunk 口 $\rightarrow$ 带着标签飞向另一台交换机 $\rightarrow$ 对方识别标签并允许进入。离开交换机：包到达目标 Access 口 $\rightarrow$ 交换机撕掉标签 $\rightarrow$ 原始包进入目标 PC。4. 博客避坑 Tip两头一致：Trunk 链路的两端配置必须对称。如果左边允许 VLAN 10，右边没写，包还是过不去。不要混用：永远不要尝试用 Access 口去连接两台需要透传多个 VLAN 的交换机，那会导致流量只能在一个 VLAN 里“打转”。
 
特性,Access 端口,Trunk 端口
连接对象,电脑、打印机（终端）,交换机、路由器、防火墙
VLAN 数量,仅限 1 个,多个（可手动指定列表）
发往对端时,撕掉 VLAN 标签（解封装）,保留 VLAN 标签（封装）
默认 VLAN 1,默认属于 VLAN 1,默认允许 VLAN 1 通过

### 补充：Hybrid 模式
在华为设备上输入 display this 时，可能会看到有些口默认是 hybrid。

Hybrid：这是华为特有的“混合模式”，它既能像 Access 一样接 PC，也能像 Trunk 一样接交换机，非常灵活。
